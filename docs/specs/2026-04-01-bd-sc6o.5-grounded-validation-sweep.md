# bd-sc6o.5 Grounded Validation Sweep

## Scope

This run validates the hardened substrate contract from `.1/.2/.3` against four
real URLs and records machine-checkable outcomes for:

- ingestion truth (`metadata.ingestion_truth`)
- trust classification (`trust_tier`, `trust_host_classification`)
- promotion state (`captured_candidate|durable_raw|promoted_substrate`)
- usefulness (`analysis_useful`, `moat_useful`)

## Repro Command

From repo root:

```bash
cd backend
poetry run python scripts/substrate/run_grounded_validation_sweep.py
```

When local `DATABASE_URL` is unavailable, run in Railway context:

```bash
railway run -p 1ed20f8a-aeb7-4de6-a02c-8851fff50d4e -e dev -s backend -- \
  sh -lc 'cd /tmp/agents/bd-sc6o.5/affordabot/backend && poetry run python scripts/substrate/run_grounded_validation_sweep.py'
```

Report output:

- `backend/scripts/substrate/artifacts/bd-sc6o.5_grounded_validation_report.json`

### What worked

- Official PDF agenda was durably captured and correctly stayed `durable_raw`
  because it was not yet retrievable.
- Official code page stayed in `durable_raw` via shell/index deny path.
- Third-party page stayed out of trusted promotion paths (`captured_candidate`).

### What remains weak

- The official HTML meeting-detail case did not reach analysis-useful state in
  this run (`durable_raw`, reason `unclear`) and showed `vector_upsert_failed`.
- Sweep logs showed vector dimension mismatch warnings (`expected 4096, not
  1536`), which can suppress `retrievable=true` and reduce promotion confidence
  for HTML captures.

This is kept explicit in the JSON report instead of being normalized away.
